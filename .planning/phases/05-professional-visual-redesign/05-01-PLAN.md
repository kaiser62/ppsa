<objective>
Establish the shared design-system foundation (spacing scale, typography scale, elevation, icon
sprite, touch-target sizing, brand-consistent header) in the single `<style>` block and `<body>`
of `docker/webui/app/static/index.html`, per UI-SPEC.md. This plan does NOT touch tab-specific
markup (Dashboard/Controls/Firewall/Backup/Wi-Fi content) — it only builds the reusable tokens and
base component rules that Plans 05-02/03/04 will apply to that markup. Every later plan in this
phase depends on this one because they consume the tokens/classes/sprite defined here.

Purpose: give every subsequent plan a single, already-tightened set of CSS variables and shared
component classes to apply, so no plan re-derives spacing/typography ad hoc (UI-02: one shared
system, not per-tab styling).
Output: Updated `<style>` block in index.html with new tokens/classes; new SVG icon sprite in
`<body>`; header brand treatment upgraded; global touch-target fix for `.nav button`/`.btn`.
</objective>
