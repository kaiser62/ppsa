<objective>
Add save-file-only backup and safe restore to the PPSA WebUI backend: a fast archive of
just the Palworld SaveGames directory (no palworld stop), and a restore flow — from an
on-box archive or an uploaded file — that validates the archive, snapshots the current
save as a safety net, stops palworld, replaces SaveGames, and restarts palworld.

Purpose: Give users a lightweight per-session save snapshot/restore path distinct from
the heavy nightly offen full-volume backup, without ever risking silent data loss.
Output: Three new protected endpoints in docker/webui/app/main.py, a validation helper,
two new _run_docker subcommands, and a compose mount change making /backups writable
from the webui container.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> POST /api/backup/restore-upload | Untrusted user-supplied .tar.gz archive crosses into the webui container's filesystem and, on validation pass, is extracted over the live Palworld save |
| Browser -> POST /api/backup/restore | Authenticated user selects an on-box filename by string; must not escape BACKUP_DIR |
| webui container -> ppsa-palworld container | Docker SDK stop/start calls, scoped to the docker.sock mount already present |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Tampering | POST /api/backup/restore-upload (tar path traversal) | critical | mitigate | _validate_save_archive rejects any archive member with an absolute path or `..` segment BEFORE extraction is ever attempted; extraction only proceeds after this check passes |
| T-03-02 | Tampering | POST /api/backup/restore (filename traversal) | high | mitigate | Resolve requested filename via `Path(filename).name` and reject (400) if it differs from raw input, matching the existing remove_mod `.name`-strip pattern |
| T-03-03 | Denial of Service | POST /api/backup/restore (destructive replace) | critical | mitigate | D-03 sequence: validate archive structurally BEFORE stopping palworld or touching SaveGames; always take a pre-restore safety snapshot before extracting; always attempt to restart palworld even on extraction failure; never delete the safety snapshot on failure |
| T-03-04 | Repudiation | Both restore endpoints | low | accept | Existing JWT auth (`require_auth`) attributes the action to an authenticated admin session; no separate audit log exists for other admin actions (mods, firewall) either — consistent with current app posture |
| T-03-05 | Information Disclosure | Uploaded archive temp file at /tmp | low | mitigate | Temp file removed via `unlink(missing_ok=True)` in a finally block immediately after use, matching install_mod's existing cleanup pattern |
| T-03-06 | Elevation of Privilege | compose mount change ../backups:/backups:ro -> rw | medium | accept | Required for save-file backup and safety snapshots to be written from webui; scope is unchanged (webui already has broader rw mounts: /etc/ppsa, palworld_data, .env) so this does not raise the container's overall privilege ceiling |
| T-03-SC | Tampering | No new package-manager installs in this plan (stdlib tarfile/shutil only) | n/a | n/a | Package Legitimacy Gate not applicable — no new npm/pip/cargo dependency introduced |
</threat_model>
