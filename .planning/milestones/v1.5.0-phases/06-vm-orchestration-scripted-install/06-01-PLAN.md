<objective>
Create the new standalone orchestrator script `scripts/ppsa-installer-e2e.py` with its
foundational layer: CLI argument parsing, VirtualBox VM lifecycle automation (create,
configure, boot headless, destroy/cleanup — all via `VBoxManage` subprocess calls), and
the pre-boot WireGuard identity collision safety check (NET-01) that must run before any
VM is ever booted.

This plan does NOT implement the TUI keystroke driving or `.installed` completion
polling — those are Plan 02, which extends this same file. This plan's job is to stand
up a runnable, testable skeleton: a script that can create/boot/destroy a VirtualBox VM
unattended, and that refuses to proceed if doing so would collide with the live
production WireGuard identity.

Purpose: VM-01 (unattended VirtualBox lifecycle) and NET-01 (pre-boot safety check +
NetBird-enrollment-timeout tolerance groundwork) are foundational — every later phase
(7, 8) depends on a working, safe VM lifecycle existing first.

Output: `scripts/ppsa-installer-e2e.py` — a runnable Python 3.12 stdlib-only script that
supports `--help`, and whose `--dry-run`-style unit-testable pieces (safety check, VM
lifecycle wrapper) can be exercised without a real installer ISO.
</objective>

       flag added in this task). If the env var and CLI flag are both unset, the

       `(safe=True, reason="no hub credentials provided; skipping live-check, proceeding
       on trust that WireGuard is disabled by default")`, because PPSA test images ship
       with `PPSA_WG_ENABLED=false` by default (CLAUDE.md constraint) and the check is a
       defense-in-depth safety net, not a hard requirement to run at all. This must be a
       graceful, non-fatal skip — printed as a clear WARNING to stderr, never silently
       swallowed.

       When credentials ARE provided: `urllib.request.urlopen()` the session POST with a
       hard `timeout=timeout` (bounded per this plan's must_haves — NEVER an unbounded
       call), extract the `Set-Cookie` header via `http.cookiejar.CookieJar` +
       `urllib.request.build_opener(HTTPCookieProcessor)`, then GET `/api/client`, parse
       JSON, find the entry where `name == WG_IDENTITY_PEER_NAME`, read
       `latestHandshakeAt` (ISO8601 or epoch-ms per the wg-easy v15 API — inspect the
       skill's documented fields; if the format is ambiguous, accept both epoch-ms int
       and ISO datetime string, and treat `None`/absent as "never handshaked = safe").
       Compute `age = now - handshake_time`. If `age < WG_HANDSHAKE_STALE_SECONDS`:
       return `(safe=False, reason=f"Real server 'ppsa-server' has a live handshake
       {age:.0f}s ago on the hub. Booting the test VM now risks stealing the shared
       10.8.0.2 identity. Pass --skip-identity-check to override at your own risk, or
       confirm the real box is offline first.")`. Otherwise return `(safe=True,
       reason=...)`.

       On ANY network error (`urllib.error.URLError`, `TimeoutError`, JSON decode
       failure, missing expected keys): catch it, do NOT crash the whole script — return
       `(safe=True, reason=f"hub check failed ({error}); could not verify, proceeding
       cautiously")` with a WARNING printed to stderr. This matches PITFALLS.md's
       guidance that this check is a safety net, not a hard network dependency — a hub
       outage must not brick every future test run.

    3. Wire this into `main()`: after arg parsing but BEFORE `create_vm()`/`boot_vm()`

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01 | Information Disclosure | check_wg_hub_identity_safe (hub password) | high | mitigate | Password sourced only from env var `PPSA_WG_HUB_PASSWORD` or CLI flag, never hardcoded or logged; verbose/debug output must not print the raw password |
