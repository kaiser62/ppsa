<objective>
Make the PPSA WebUI dashboard honest: show the real game version, distinguish "starting" from "broken" on fresh boot, render explicit empty states instead of silent blanks, and stay usable with clear status banners when upstream calls fail.

Purpose: The dashboard is the first thing users see. Current behavior shows "-" for version, empty/zeroed fields during server startup, and silently swallows errors. This destroys trust and makes the appliance look broken when it's just booting. Fixing these five gaps makes the appliance feel reliable even during initialization.

Output: Updated `main.py` (dashboard endpoint with container state detection, version log-parsing fallback) and `index.html` (state-aware rendering, alert banner, explicit empty states).
</objective>
