### Fix #1: `ppsa-wifi-onboard.service` is now installed and enabled
**Status:** ✅ WORKING

Before (v1.1.0): `systemctl list-unit-files ppsa*` showed only `ppsa-install.service`.
After (v1.1.1): Shows both `ppsa-install.service` and **`ppsa-wifi-onboard.service`** (enabled).

The service was installed by `install.sh` Step 4 (`[4/7] Installing PPSA Wi-Fi onboarding service...`). The service ran on first boot and wrote to its log:
```
=== ppsa-wifi-onboard Sat Jun 27 07:23:36 PM UTC 2026 ===
[19:23:36] No Wi-Fi hardware detected. Exiting.
```
This is the expected behavior on a VM with no wireless hardware — the script detects no Wi-Fi and exits silently. On a real laptop, it would start the PPSA-Setup hotspot.

## 🔬 Wi-Fi Hotspot End-to-End Test — NOT POSSIBLE IN VM

Attempted to pass the host's Intel AX200 Wi-Fi to the VM for end-to-end hotspot testing. **Not possible on this host.** Three blockers:

1. **VirtualBox 7.2 on Windows has no PCI passthrough support.** The `VBoxManage list hostpci` command was removed; only USB passthrough is available. The only Wi-Fi device on the host (Intel AX200) is PCI, not USB.

2. **Hyper-V is running on the host** (required for WSL/Docker). VBox cannot access VT-x directly and reports `VirtualizationFirmwareEnabled = False`. PCI passthrough requires VT-d/IOMMU which VBox can't use while Hyper-V is active.

3. **No USB Wi-Fi adapter is connected.** The host has USB mouse, keyboard, Bluetooth, and storage, but no USB Wi-Fi dongle. The Intel Bluetooth companion (`VID_8087&PID_0029`) is on USB but is the BT half of the AX200, not a standalone Wi-Fi adapter.

**To test the hotspot end-to-end, one of these is required:**

- **Physical hardware test**: Boot v1.1.1 on a real laptop (or USB stick) with built-in Wi-Fi. The code paths are verified (service installs, runs, detects hardware correctly), but the actual hotspot broadcast + captive portal needs a real radio.
- **USB Wi-Fi adapter**: Plug any USB Wi-Fi dongle into the host (RTL8188/RTL8812 chipsets are best — well-supported in Linux). VBox USB passthrough works. Cost: ~$10-15.
- **Different hypervisor**: VMware Workstation or Hyper-V can do PCI passthrough with proper setup.

**What IS verified in VM:**
- ✅ `ppsa-wifi-onboard.service` installs and enables on first boot
- ✅ Service runs at first boot, detects no Wi-Fi hardware, exits cleanly (script log: "No Wi-Fi hardware detected. Exiting.")
- ✅ All Wi-Fi firmware present (ath10k, ath11k, ath12k, iwlwifi-*, etc.) for real hardware
- ✅ All Wi-Fi tools installed (nmcli, hostapd, wpa_supplicant)
- ✅ API endpoint returns 200 with `no_wifi_hardware` status (not 500)

**What requires physical hardware:**
- Actual PPSA-Setup hotspot broadcast
- Phone/laptop connecting to the hotspot

- Wi-Fi network scanning (returns real SSIDs)
- Connecting to a real Wi-Fi network via the WebUI
