## 🐛 BUGS FOUND

### 🔴 BUG #1 (CRITICAL): `ppsa-wifi-onboard.service` NOT installed
- The service file exists at `/opt/ppsa/scripts/ppsa-wifi-onboard.service` (508 bytes)
- The script exists at `/opt/ppsa/scripts/ppsa-wifi-onboard.sh` (4607 bytes)
- BUT: never copied to `/etc/systemd/system/` and never enabled
- `systemctl list-unit-files ppsa*` shows only `ppsa-install.service`
- **Impact:** The hotspot fallback never starts on first boot. Users can't connect to a real Wi-Fi via the WebUI on a fresh image.
- **Fix:** Add 3 lines to `scripts/install.sh`:
  ```bash
  cp /opt/ppsa/scripts/ppsa-wifi-onboard.service /etc/systemd/system/ppsa-wifi-onboard.service
  systemctl daemon-reload
  systemctl enable ppsa-wifi-onboard.service
  ```

### 🟡 BUG #2: Disk 92% Full (8GB image too small)
- `/dev/sda2: 7.6G, 7.0G used, 209M free (98%)`
- Palworld game alone is 3.8GB
- 5.3GB system + 3.8GB game + backups = no headroom
- **Impact:** First full backup will fail. Updates will fail.
- **Fix:** Increase image size from 8GB to 12GB in `build-live-usb.sh`:
  ```bash
  # Change IMG_SIZE="8G" to IMG_SIZE="12G"
  ```

### 🟡 BUG #3: Wi-Fi hotspot start returns 500
- `POST /api/wifi/hotspot/start` → `{"detail":"hotspot start failed: "}`
- **Root cause:** No Wi-Fi hardware in VM, so `nmcli device wifi` returns empty list, hotspot creation fails.
- **Impact:** On a real laptop with Wi-Fi, this would likely work. Need to test on hardware.
- **Not really a bug** — expected on hardware without Wi-Fi.
